# 定制加购与编辑器改造（v1）

## 1) 目标能力拆解（基于公开行业页面）

- 商品层：一个类目页下有多个定制 SKU（标题、价格、品类、材质厚度等信息强绑定）。
- 加购层：用户先选规格（如厚度、尺寸、工艺）再进入定制或直接购买，SKU 与规格组合保持一致。
- 定制层：支持图片上传、图层编辑、自动裁切/预览、印刷清晰度提示（DPI）。
- 人群层：同一套商品同时服务
  - B端客户：批量定制（数量、交付、稳定履约）
  - C端粉丝：单件快速定制（低门槛、即看即下单）

## 2) FUNDAMENTAL 现状到改造后的数据流

### 旧流程（改造前）

`商品详情 -> 定制页保存 customization -> checkout 下单`

问题：没有统一购物车，标准 SKU 加购和定制加购分离，不利于批量场景。

### 新流程（本次改造后）

`商品详情(选规格/SKU) -> 加入购物车`
`定制编辑器(保存 customization) -> 加入购物车`
`购物车统一查看标准款+定制款`

对应技术模块：

- `lib/cart-store.ts`：购物车中台（本地 JSON store）
- `app/api/cart`：购物车 API（增删改查）
- `app/cart`：购物车页面
- `ProductDetailClient`：产品 SKU 加购
- `CustomizeEditor`：定制稿加购

## 3) 与 demo 目标的关系

- B端批量：购物车支持数量调整，后续可直接扩展“批量导入 SKU + 数量”。
- C端单件：从定制页保存后可一键入购物车，再统一下单。
- 编辑器能力：保留并强化“上传图片 + 文本 + 颜色 + DPI 提示 + 定制稿绑定加购”主链路，下一步可接入 Canvas 级图层编辑。

## 4) 下一步建议（v2）

- 增加“SKU 组合合法性校验”与“规格映射表”。
- 编辑器补充：拖拽缩放、旋转、裁切框、出血线、安全线。
- 购物车到订单：支持一次性提交多行（B端批量下单）。
- 增加日本语/韩语文案切换与币种/税费策略。
